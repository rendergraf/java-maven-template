Si el formato del proyecto no se maven o no se envía el proyecto entero la nota del examen será un 0.

→ No se corregirá código que esté comentado (que no se ejecute).
→ No se corregirá código con errores de sintaxis o errores de ejecución.
→ No se pueden utilizar conceptos que no se hayan dado en clase.
→ Se valorará que el nombre de las variables, métodos, packages y clases sean definitorios y hagan el código más comprensible. Es imprescindible organizar el código en clases y éstas en packages. El código debe organizarse en métodos.
→ Si se utiliza bucles infinitos se invalidará la totalidad del examen.
→ No se puede utilizar break (excepto en el switch), continue, exit ni bucles infinitos. Tampoco return sin valor ni funciones lambdas (que no se han dado en clase).
→ Cuando el enunciado muestre ejemplos de ejecución debe replicarse el formato de entrada y salida mostrado.
→ No se puede utilizar ningún tipo de IA ni Internet para resolver el examen.
→ Se debe añadir un comentario justificando la elección de cada colección que utiliza en su solución.
→ Es imprescindible crear y hacer uso de Exceptiones propias tal y como se ha enseñado en clase y gestionar correctamente todas las excepciones.
